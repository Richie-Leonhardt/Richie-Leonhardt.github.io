git连接账户和邮箱的设置

一般默认在用户的目录下启动了git-bash，我这边是Administrator目录下，设置账户和邮箱，下面的两个参数修改成自己的账户名和邮箱

1
2

$ git config --global user.name "Richie Leonhardt"
$ git config --global user.email example@163.com

设置成功后这个目录下应该会有.gitconfig的文件，实际上，在转移到一台新的机器上后考研直接拷贝这个存有账户和邮箱的文件到相应的目录下

使用git生成ssh秘钥

1

$ ssh-keygen -t rsa -C "example@163.com"

三次回车之后，在目录下生成一个.ssh文件夹，里面有公钥id_rsa.pub和私钥id_rsa

将公钥内容添加到GitHub的账户(此处省略步骤)

添加key到agent

1

$ ssh-add ~/.ssh/id_rsa

为了确保每次打开git-bash都加载agent并且添加了私钥，在这个目录下新建.profile的一个文件，内容为以下 :

env=~/.ssh/agent.env

agent_load_env () { test -f "$env" && . "$env" >| /dev/null ; }

agent_start () {
    (umask 077; ssh-agent >| "$env")
    . "$env" >| /dev/null ; }

agent_load_env

# agent_run_state: 0=agent running w/ key; 1=agent w/o key; 2= agent not running
agent_run_state=$(ssh-add -l >| /dev/null 2>&1; echo $?)

if [ ! "$SSH_AUTH_SOCK" ] || [ $agent_run_state = 2 ]; then
    agent_start
    ssh-add
elif [ "$SSH_AUTH_SOCK" ] && [ $agent_run_state = 1 ]; then
    ssh-add
fi

unset env

重启一下git-bash

这个时候发现已经加入了私钥，那么最后测试一下连接一下就行了，返回1表示连接成功

1

$ ssh -vT git@github.com

输入yes确认，可以发现.ssh目录下出现了一个新文件known_hosts，连接成功。

至此环境已经配置好了，之后每次git-bash启动之后都会加载ssh agent并且添加.ssh目录里的私钥了，每次提交到GitHub都不用再输入用户名和密码了

换机器的时候快速配置环境

复制.ssh目录下的公钥和私钥文件，和用户目录下的.gitconfig和.profile文件，一共四个文件连带.ssh文件夹打包一下拷到U盘或者QQ里，然后解压到相应的目录，最后启动git-bash测试一下连接就行了

1

$ ssh -vT git@github.com

使用ssh key时本地仓库与GitHub仓库相关联

另一点需要注意的是，使用ssh连接GitHub时，应该以下面这样设置而不是https设置，否则会出现已有ssh仍然需要登录GitHub输入账号和密码的问题，因为用的是https的连接。

1
2

$ git remote remove origin
$ git remote add origin git@github.com:Username/Your_Repo_Name.git

除此之外，修改成ssh连接之后还需要重新设置track branch，在命令行中等号中间不要有空格，不是在写程序

1

$ git branch --set-upstream-to=origin/master master
